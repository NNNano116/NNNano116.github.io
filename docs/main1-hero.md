## 1. 개요

전체 뷰포트를 채우는 **부유하는 3D 구체 클러스터**를 배경으로, 가운데 큰 타이틀·태그라인을 얹은 랜딩.
구체는 자체 구현한 **경량 물리 루프**(외부 물리 라이브러리 없음)로 움직이며, 마우스/클릭에 실시간 반응한다.

- **유형**: 인터랙티브 히어로(데모/메인 후보). 정적 전제 유지(클라이언트 only) → [CLAUDE.md §1](../CLAUDE.md).
- **렌더**: `three.js`(WebGL) 3D 씬. **렌더링은 캔버스**, **타이틀·로고·메뉴·푸터는 DOM 오버레이**.

## 4. 렌더링 구조

| 요소 | 설정 | 의도 |
|------|------|------|
| `WebGLRenderer` | `antialias`, **`alpha:true`**(투명) | 배경 그라디언트는 **CSS**가 담당, 텍스트/크롬은 DOM이 캔버스 위에 |
| 톤매핑 | `ACESFilmicToneMapping`, exposure `0.86` | 하이라이트 롤오프 → 구체 **백화(흰색 뭉개짐) 방지**, 파스텔 라벤더 유지 |
| 카메라 | `PerspectiveCamera(45)`, z=18(세로 화면 보정) | 클러스터가 가운데를 적절히 채우도록 |
| 환경광 | `RoomEnvironment` GI, `environmentIntensity 0.45` | 매끈한 음영(평탄·말랑한 점토 느낌) |
| 조명 | Hemisphere(0.85) 위주 + 약한 Directional(key 0.48 / fill 0.22) | 방향광이 세면 정면이 흰색으로 떠서 약하게(헤미스피어 위주) |
| 구체 | `SphereGeometry(1,48,32)` 1개 공유 + 메시별 scale, `MeshStandardMaterial`(roughness 0.5) | 라벤더 톤 5색, 데스크탑 **32개**/모바일 22개 |

> 캔버스는 z=0 평면 기준. 포인터는 `Raycaster`로 z=0 평면에 투영해 월드 좌표로 변환한다.

## 5. 물리 모델 (프레임 루프)

각 구체는 `pos`/`vel`(THREE.Vector3)을 가지며, 매 프레임 힘을 적분한다. (할당 없는 임시 벡터 재사용)

### 5-1. 상시 힘 — 세 힘의 교차(crossfade)
세 가중치를 **서로 다른 위상의 느린 사인**으로 흔들어 번갈아 강조한다(항상 켜져 있되 강약 교차):
- `wMagnet/wFloat/wFlow = 0.7 + 0.3·sin(t·{0.12,0.1,0.08} + φ)`

| 힘 | 식(요지) | 효과 |
|----|----------|------|
| **자석(응집)** | `acc += -2.85·pos·wMagnet` | 중심으로 모으는 스프링 |
| **부유** | `acc += sin(t·…+위상)·{4.4,4.4,2.6}·wFloat` | 구체마다 위상 다른 잔잔한 떨림 |
| **흐름** | `vel += (flowAxis × pos)·0.55·wFlow·dt` | 축 둘레로 도는 소용돌이(입체 회전). **반지름 비례라 과하면 발산 → 약하게** |
| **감쇠** | `vel *= 0.18^dt` | 프레임레이트 독립. 에너지 누적/발산 방지 |
| **경계** | `|pos|>7.6` 이면 되돌리고 `vel*0.82` | 화면 밖 이탈 방지(탄력) |

### 5-2. 상호작용 힘
| 입력 | 동작 |
|------|------|
| **커서 반발(상시)** | 반경 `r+4.2`(드래그 시 +1.6) 안에서 `f²·120·dt`로 밀어냄(가까울수록 급증=통통) |
| **드래그 휩쓸기** | 커서 화면속도를 평활화 → 진행 방향(`sweepDir`)으로 끌고 감. **드래그(버튼)면 ×1, 호버는 ×0.45**. 멈추면 빠르게 감쇠 |
| **클릭 버스트** | `pointerdown` 시 반경 `r+6` 안 구체를 방사상으로 `(1-d/R)·23` 임펄스 |

### 5-3. 구체 간 상호작용 (O(n²), n작아 충분)
- **분리**: 겹치면(`dist < r+r+gap`) 위치를 절반씩 밀어 떼어놓음.
- **표면장력(이웃 응집)**: 약간 떨어진(`< min+2.4`) 이웃을 끌어당김. `cohK = 5.2·mNorm` — **모일 때만(mNorm↑)** 강해져 디테일하게 nestle.
- **동적 간격**: `gap = 0.46 − 0.27·mNorm` — 모일수록 좁혀 빈틈없이, 흩어질 땐 벌려 또렷이.
  - `mNorm = clamp((wMagnet−0.4)/0.6, 0, 1)` = "지금 얼마나 모이는 국면인가".

## 6. 속도 임계값 기반 반발 (떨림 제거의 핵심) ⭐

가운데로 뭉쳤을 때 **부들부들 떨림(버즈)**을 없애기 위해, 충돌 반발을 **접근속도에 연동**한다.

- 접촉 시 접근속도 `approach`(서로 다가오는 정상속도)를 구하고, 반발계수 `e`를 다음처럼 복구:
  - `approach ≤ vLow(1.6)` → **e = 0** (완전 비탄성 = **속도 흡수 → 정지**, 떨림 없음)
  - `vLow ~ vHigh(4.5)` → e 를 0→`eMax(0.9)` **선형 복구**
  - `approach ≥ vHigh` → e = 0.9 (탱탱)
- 임펄스: 등질량 충돌식 `jimp = (1+e)·approach·0.5` → 상대 정상속도를 `−e·approach`로.
  - **e=0 이면 상대 정상속도를 0으로 흡수**(미끄러져 통과 X, 튕김 X) → 가운데 모인 클러스터가 잔진동 없이 가라앉는다.
  - 빠른 입력(클릭/빠른 드래그)으로 속도가 커지면 e가 살아나 **다시 통통**.

> ⚠️ 주의: 이 임펄스식에서 반발계수를 **0 미만/흡수 부족**으로 두면 접근속도가 흡수되지 않아 위치보정과 싸우며 **떨린다**. 반드시 `e=0`이 "완전 흡수"가 되도록(위 식) 유지할 것.

## 7. 안정성 / 검증 (Playwright 직접 관찰)

`@playwright/mcp`로 실제 페이지를 조작하며 **시간 경과에 따른 평균/최대 속도·클러스터 퍼짐(spread)**을 측정해 튜닝했다.

- **퍼짐 고정**: 한 사이클(~57s)에서 spread **2.91–3.65**로 유지(에너지 발산 없음). 흐름 힘 약화 + 감쇠 강화로 해결.
- **차분 국면**(부유/흐름 약): 평균속도 ~0.72, 최대 ~1.2 → 모두 `vLow(1.6)` 미만이라 **모든 접촉 e=0 → 떨림 0**. 속도 시계열이 매끈(스파이크 없음).
- **상호작용 복구**: 정지 ~1.0 → **클릭 시 ~22 / 빠른 드래그 ~14** 로 튀어, 임계값 위에서 반발이 정상 복구됨을 확인.
